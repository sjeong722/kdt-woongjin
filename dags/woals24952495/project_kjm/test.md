# 📊 프로젝트 분석 개요

# 🚀 Subway-Pulse
### 실시간 지하철 지연 분석 서비스

### 1. 프로젝트 기본 정보

- **프로젝트명**: **[MetroPulse]** 실시간 데이터 기반 지하철 노선별 지연 지점 가공 및 분석
- **팀원**: 김재민, 이찬구

---

### 2. 주제 선정 및 배경


- **해결하고자 하는 문제**: 
    - 승객의 최대 스트레스 요인인 '지하철 연착' 정보의 부재 해결.
    - 기존 서비스는 개별 열차 위치만 알려줄 뿐, 노선 전체의 **전반적인 지연 정체 구간** 정보는 부재함.
- **왜 이 문제가 중요한가?**: 
    - 경춘선, 폭설 시의 지하철 마비 등 특정 상황에서의 지연은 사회적 비용 발생.
    - "단순 위치"가 아닌 "지연 정도"를 정량화하여 제공하는 것이 핵심 가치.
- **핵심 질문/목표**:
    1. **실시간 정보 동기화**: 시간표(Static) 대비 실시간 위치(Live) 오차 분석.
    2. **지연 기준 수립**: 단순히 불일치하는 것이 아니라 **1분 이상의 오차**를 유의미한 지연으로 정의 (승하차 오차 제외).
    3. **자동화 파이프라인**: 1분 단위로 실시간 지연 현황을 자동으로 수집 및 시각화.

---

### 3. 데이터 기술 및 전처리

- **데이터 파이프라인 (End-to-End)**
    - **Collection**: Python 기반 실시간 데이터 수집 및 정제 (`kjm_1.py`).
    - **Pipeline**: Git & Airflow를 활용한 워크플로우 자동화.
    - **Storage**: Supabase 클라우드 DB 적재.
    - **Visualization**: Redash를 활용한 SQL 기반 시각화.

- **스마트 키 로테이션 (Smart Key Rotation)**
    - **한계**: API 키 1개당 일일 1,000회 제한 (13개 노선 수집 시 약 1.3시간 내 소진).
    - **해결**: 12개의 API 키 리스트화 및 순차 자동 전환(Relay) 알고리즘 구현.
    - **효과**: 주요 운행 시간(05~25시) 동안 끊김 없는 데이터 수집 달성.

- **전처리 및 표준화**
    - **변수 선별**: 13개 변수 중 분석 핵심 8개 선별 및 직관적 영문명 통일.
    - **시간 데이터 정밀화**: `Pendulum` 활용, 한국 표준시(Asia/Seoul) 즉시 동기화.
